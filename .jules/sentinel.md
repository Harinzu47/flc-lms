# Sentinel Journal - Critical Security Learnings

## 2026-07-03 - [Path Traversal in Secure Downloads]
**Vulnerability:** The submission file download controller loaded files directly using `Storage::disk('local')->download($submission->file_url)`. Although `file_url` was stored in the database, if any entry contained `..` relative path segments (e.g. from data tampering or admin import errors), it could allow arbitrary file reading outside of the submissions folder.
**Learning:** Even when reading paths from database records, defense-in-depth requires explicitly validating that the path does not traverse out of the intended directory.
**Prevention:** Always sanitize/verify all path strings before loading them into file operations using simple string check `str_contains($path, '..')` to completely block path traversal vectors.

## 2026-07-03 - [Missing Rate Limiting on Auth Endpoints]
**Vulnerability:** Authentication mutation endpoints such as forgot-password, reset-password, confirm-password, and update password had no rate limiting applied. This left them vulnerable to automated brute-forcing, resource exhaustion, and email flooding.
**Learning:** Default routes generated by scaffold templates might not include throttling by default.
**Prevention:** Audit all POST/PUT/PATCH endpoints in `routes/auth.php` and apply `throttle:6,1` (or tighter limits like `throttle:3,1` for email triggers) to protect resource-sensitive mutations.
